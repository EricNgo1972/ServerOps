# ServerOps Context Pack

I am building a .NET 8 Blazor Server application called **ServerOps**.

## Purpose

ServerOps is a lightweight DevOps platform to manage applications on a single host (Linux or Windows).

It can:

* Discover and control services (start/stop/restart)
* Deploy apps from GitHub releases
* Manage Cloudflare tunnels and domains
* Perform one-click deployment (deploy + expose)
* Track deployment history and support rollback
* Store logs per deployment

---

## Architecture

Clean layered architecture:

* **Domain**: entities, enums (AppInstance, DeploymentRecord, etc.)
* **Application**: abstractions + orchestration services
* **Infrastructure**: implementations (host, deployment, cloudflare, storage)
* **Web**: Blazor Server UI (no heavy UI frameworks)

No external dependencies unless absolutely necessary.

---

## Completed Backend Modules

* Module 1–4: Host service + service control (Linux/Windows)
* Module 5–7: App topology and filtering (company apps only)
* Module 8: Cloudflare tunnel + DNS automation
* Module 9: Deployment pipeline (download → extract → swap → start → verify)
* Module 10: Deployment history + rollback (JSON-based, safe)
* Module 11: One-click deploy (GitHub → Deploy → Domain → Tunnel)

---

## Key Concepts

* **App** = a managed service deployed on the host
* **Deployment** = one version install attempt
* **OperationId** = deploymentId used for logs
* **Exposure** = mapping app → domain via Cloudflare tunnel
